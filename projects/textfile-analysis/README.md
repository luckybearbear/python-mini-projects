# textfile-analysis 文本字符统计

## 项目背景

### 需求

读取控制台输入字符串，按行输出 `字符,次数`

### 拓展改造（必做）

1. 支持读取本地txt文件统计，而非仅控制台输入；
2. 过滤空格、换行符，不计入统计；
3. 按出现次数从高到低排序输出。
4. 中文分词


### 当日目标

熟练字典计数、文件读写、字符串清洗。

## 总结

### 1. 导入模块是什么意思？
- `sys`：负责读取你在黑框里输入的文件路径，实现命令行传文件
- `collections.Counter`：专门做计数，比自己用字典 `get()` 更简单
- `string`：自带英文标点，不用自己手动敲一堆符号
- `jieba`：第三方中文分词库，**中文必须靠它拆分词语**，英文直接空格分割就行

### 2. 命令行参数判断逻辑
运行命令：`python 脚本名.py test.txt`
此时 `sys.argv = ["脚本名.py", "test.txt"]`，长度是2。
如果你少输文件，长度=1，程序会打印使用方法并退出，不会崩溃。

### 3. 文件读取 + 异常捕获
```python
try:
    open(...)
except FileNotFoundError:
    print("文件不存在")
```
作用：如果路径写错、文件删掉了，不会直接崩溃报一大段红色报错，而是友好提示。

### 4. 标点清洗核心
`str.maketrans("", "", punc)` 是批量删除字符的工具。
原文 `我，爱Python！` → 清洗后变成 `我爱Python`，标点不会被当成词语统计。

### 5. jieba分词作用
中文句子没有空格，普通 `split()` 会把一整段当成一个词。
比如 `今天天气很好`
不用jieba：`["今天天气很好"]`（1个词，完全错误）
用jieba：`["今天", "天气", "很", "好"]`（正确拆分词语）

### 6. 过滤空白词语
分词偶尔会分出空字符串、纯空格，`if w.strip()` 直接扔掉无效内容，不参与统计。

### 7. 统计数值计算
- `sum(cnt for _, cnt in word_items)`：把所有词语出现次数加起来 = 全文总词数
- `len(counter)`：Counter内部自动去重，长度就是独一无二词语数量

### 8. 输出前5高频词
`word_items = counter.most_common()` 已经从高到低排好序，`[:5]` 切片只拿前5条打印。

# 运行示例
